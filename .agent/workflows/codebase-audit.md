# 전체 코드베이스 점검 (Audit)

전체 작업을 총괄하여 대규모 수정 소요나 결함을 정밀 분석할 때 가동하는 전용 코드 점검기입니다.

⚠️ **주의**: 사용자가 직접 명시적으로 `/codebase-audit` 이나 이에 준하는 "점검" 관련 지시를 내리지 않는 이상 어떠한 발화 맥락에서도 선제 작동 및 자동 호출 제안을 하지 마십시오.

## Workflow

점검 절차는 다음의 8개 페이즈(Phase)로 점진적 확장 및 순차 진행됩니다.

- **Phase 0 (범위 선택)**: 점검 대상이 될 프로젝트(혹은 공통 폴더) 구역 스케일 범위를 가늠/설정.
- **Phase 1 (결함)**: 눈치채지 못했던 명백한 문법 실수/런타임 결함 요소 판정.
- **Phase 2 (미사용 코드)**: 데드 코드, 쓰이지 않는 임포트나 함수 덩어리 적발.
- **Phase 2.5 (패턴 위반)**: 기존의 반복 검증 패턴 규칙 등 필수 아키텍처 가이드라인 침해 내역 탐지.
- **Phase 3 (개선)**: 리팩토링이나 최적화를 수행하면 더 좋아질 구조적 가치 제시.
- **Phase 4 (UX)**: 사용자 시점 모바일/웹 응답성 및 접근성 약점 진단.
- **Phase 5 (미구현 UI)**: 겉만 있고 데이터 바인드나 작동이 배제되어 있는 목업 단계 UI 식별.
- **Phase 6 (Cross-project 시너지)**: 타 하위 레포 연관된 부분에서의 상충 요인 점검.
- **Phase 7 (환경설정)**: 기본 설정 환경파일(tsconfig 등)에서의 느슨한 조이기.

### 산출물 경로 표기

- 출력 결과 리포트 타겟 디렉토리: `common/docs/audit/{date}_codebase-audit.md`
- 취약점의 판정 Severity 등급 명기 (critical / warning / info 단위)
