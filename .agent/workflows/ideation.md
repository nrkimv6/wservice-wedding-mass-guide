# 신규 아이디어 발굴 (Ideation)

하위 프로젝트들이나 새로운 방향에 대한 아이디어를 자극하고 발굴하여 새로운 기획으로 엮어 냅니다.

⚠️ **주의**: 명백한 호출 지시(`/ideation` 등) 없이 어떠한 연상 발화만으로도 이를 사용자에게 선행, 선제 자동 제안으로 내놓지 마십시오.

## Workflow

아이디어 창출 절차는 아래 단계 페이즈(Phase)를 따라 시도합니다:

- **Phase 1 (UX 향상)**: 현재 보유 기능에서의 추가적인 불편/개선 요소 시나리오 발굴
- **Phase 2 (기능 확장)**: 기존 서비스 구조에 부가가치를 창출하는 방향 제언
- **Phase 3 (신규 기능)**: 백지에서 출발하여 도입 가능한 기능 도출
- **Phase 4 (DX 개선)**: 개발 단계에서의 생산성, 체계화 등의 요소 보완
- **Phase 5 (Cross-project 연계)**: 여러 개의 하위 앱들을 종합 시너지로 묶을 아이템
- **Phase 6 (트렌드 반영)**: 최신 외부의 API, 트렌드, 디자인 시스템 추세 반영 방향 구상

### 제한 사항

- **공통 패키지**나 강제적인 분리 레포에 대한 제안은 금지되어 있습니다. 모노레포 등의 선언 없이 이론상 `_sample` 등지를 참조하는 방향성 수반을 고려해야 합니다.
- **자동 선 등록 불가**: 시스템이 사용자의 단독 승인 없이 해당 발제 문서의 등록 제안을 임의로 수행하는 것 역시 불가합니다.

### 산출물
- 보고서 리포트 경로: `common/docs/idea/{date}_ideation-report.md`
- 우선순위 지정: 각 발제 항목별 P0 ~ P3 등급으로 나누어 마킹합니다.
