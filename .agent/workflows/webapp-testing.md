# 웹앱 검증 및 빌드 테스트

해당 어플리케이션의 타입 검사, 빌드 호환성 및 UI 패턴들을 종합 점검하기 위한 테스트 워크플로우입니다.

## Workflow

### 1단계: 타입 체크

프로젝트 디렉토리 루트 상에서 `npm run check` (SvelteKit이나 Astro의 Type Validation 구문 이용)를 가동합니다.
오류 발생 시에는 해당 파일 및 출력 라인을 사용자 리포트에 표기합니다.

### 2단계: 빌드 프로세스 가동

이어서 `npm run build` 스크립트를 거칩니다. Cloudflare Workers 배포 환경에 적응하도록 구성된 adapter-cloudflare가 구동되어 빌드 파일 덩어리들이 제대로 `build/` (스벨트), `dist/` (아스트로) 내역에 출현하는지 검사합니다.

### 3단계: UI 검증 체크리스트

다음 6가지 UX 핵심 요소가 고려되었는지 점검합니다:
1. 페이지네이션 결핍: 다량의 레코드 표출 리스트일 시 필수 유무 확인
2. 이벤트 버블링 단속: 내부 요소를 갖는 영역 내 요소는 `e.stopPropagation()` 중첩 클릭 발생 문제 조사
3. FK 이름 관계성: 의미없는 참조 숫자나 아이디만 노출되지 않았는가 조사 (테이블 상)
4. 벌크 액션 툴바: 선택 행위가 존재할 시 벌크 처리 기능 제공
5. 전체 선택 트리거: 전체 범위 지정 기능 존재 확인.
6. 데이터 변경 이후 새로고침/자동 동기화 처리 유무.

### 4단계: 반복 패턴 위반 검사 점검

사전에 설정된 프론트엔드 코드상 4대 가이드 패턴들이 침해받지 않고 적용 중인지 모색합니다:
- **selection Set**: 복수 처리를 위해 Array가 아닌 Set 사용 (불변성 유지)
- **toast store**: 모달/에러의 전역적 발생 알림으로 시스템 Toast 기능 사용
- **local state update**: 딜레이 감쇄를 위해 즉각적인 낙관 지배적 UI 변환 사용
- **401 cooldown**: 인증 API 토큰 등이 누락되었을 시, 에러 재도전에 쿨다운 시간 삽입을 사용하여 과열 방지.
