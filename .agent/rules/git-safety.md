# Git 안전 규칙

## 절대 금지 행위

```
❌ .git 디렉토리 삭제 (rm -rf .git, Remove-Item .git 등)
❌ .git 내부 파일 수정/삭제
❌ git clean -f, git clean -fd (추적되지 않는 파일 강제 삭제)
❌ git reset --hard (커밋되지 않은 변경사항 강제 삭제)
❌ git checkout . 또는 git restore . (변경사항 일괄 되돌리기)
❌ Remove-Item -Recurse -Force (코드 디렉토리 대상)
❌ 하위 프로젝트의 .git 디렉토리 삭제/이동/수정
❌ "복구하겠다"며 기존 코드나 .git을 삭제하는 행위
```

## 강제 체크 (모든 명령 실행 전 반드시 확인)

- [ ] 이 명령이 `.git` 디렉토리를 삭제하거나 손상시키는가? → **즉시 중단**
- [ ] 이 명령이 코드 파일을 대량으로 삭제하는가? → **즉시 중단**
- [ ] `Remove-Item`, `rm`, `del`, `rmdir`에 `-Recurse`, `-Force`, `-rf` 플래그가 있는가? → **대상 경로에 .git이 포함되는지 반드시 확인**
- [ ] git 문제 "해결"을 위해 .git 삭제 후 재생성을 고려하고 있는가? → **절대 금지. 사용자에게 상황 보고만 할 것**

## git 문제 발생 시 올바른 대응

1. **현재 상태를 사용자에게 보고** (삭제/복구 시도 금지)
2. `git status`, `git log --oneline -5` 등 읽기 전용 명령으로 상태 파악
3. 사용자의 명시적 지시 없이 파괴적 명령 실행 금지
4. .git 관련 문제는 **사용자가 직접 판단**하도록 정보만 제공
