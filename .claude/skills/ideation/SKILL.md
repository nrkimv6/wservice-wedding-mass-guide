## 실행 단계

### Phase 0: 분석 범위 선택

사용자에게 질문:
- **전체 분석** (모든 카테고리, 모든 프로젝트)
- **특정 프로젝트만** — 1~2개 프로젝트 집중 분석
- **특정 카테고리만** — 아래 카테고리 중 선택

기본값: 전체 분석

---

### Phase 1: UX 개선 기회 (User Experience Enhancement)

현재 UX를 분석해서 더 나아질 수 있는 부분을 찾습니다.

**1-1. 사용자 흐름 단축 기회**
- 여러 단계가 필요한 작업 → 원클릭으로 가능한지
- 반복적인 입력 패턴 → 자동 완성/기본값 설정
- 자주 쓰는 기능인데 깊숙이 있는 것 → 상위 노출

**1-2. 피드백 강화 기회**
- 성공/완료 시 더 명확한 피드백
- 진행 상황 표시 추가 가능 여부
- 결과 미리보기 추가 가능 여부

**1-3. 개인화 기회**
- 사용자 선호도 저장 가능 여부
- 최근 사용 항목 기억
- 커스터마이징 옵션 추가

---

### Phase 2: 기능 확장 기회 (Feature Extension)

기존 기능을 확장해서 더 유용하게 만들 수 있는 방법을 찾습니다.

**2-1. 데이터 활용 확장**
- 저장된 데이터로 통계/분석 제공 가능 여부
- 데이터 내보내기(export) 기능 추가 가능 여부
- 데이터 시각화 추가 가능 여부

**2-2. 연동 기회**
- 프로젝트 간 데이터 연동 (예: activity-hub ↔ sacred-hours)
- 외부 서비스 연동 (캘린더, 알림 등)
- 공유 기능 추가 가능 여부

**2-3. 배치/자동화 기회**
- 수동 반복 작업 → 예약/자동 실행
- 조건부 자동 트리거
- 일괄 처리(bulk) 기능

---

### Phase 3: 새 기능 아이디어 (New Feature Ideas)

기존 코드 패턴과 도메인을 분석해서 새 기능을 제안합니다.

**3-1. 코드 패턴 기반 아이디어**
각 프로젝트의 핵심 기능 분석:
```powershell
# 주요 API 엔드포인트 확인
Get-ChildItem "D:\work\project\service\wtools\{project}\src\routes\api" -Recurse -Filter "*.ts"

# 주요 Store/State 확인
Get-ChildItem "D:\work\project\service\wtools\{project}\src\lib\stores" -Recurse
```

- 비슷한 패턴 반복 → 통합/일반화 가능한 새 기능
- 한 프로젝트에만 있는 유용한 기능 → 다른 프로젝트에도 적용

**3-2. 도메인 기반 아이디어**
각 프로젝트 목적에 맞는 자연스러운 확장:
- activity-hub: 활동 추적 → 목표 설정, 성과 분석
- sacred-hours: 기도 시간 → 기도 일기, 의향 관리
- memo-alarm: 메모 알림 → 반복 알림, 조건부 알림
- 등등...

**3-3. 경쟁/유사 앱 대비 분석**
- 해당 도메인의 다른 앱들이 가진 기능 중 없는 것
- 차별화 포인트가 될 수 있는 독특한 기능

---

### Phase 4: DX 개선 기회 (Developer Experience)

개발 생산성을 높일 수 있는 영역을 찾습니다.

**4-1. 개발 환경 개선**
- 빌드 속도 개선 가능 여부
- 핫 리로드 품질
- 타입 안전성 강화 기회

**4-2. 테스트 개선**
- 테스트 커버리지 부족 영역
- E2E 테스트 추가 필요 영역
- 테스트 자동화 기회

**4-3. 배포/운영 개선**
- CI/CD 파이프라인 개선
- 모니터링/로깅 강화
- 에러 추적 개선

---

### Phase 5: 프로젝트 간 시너지 (Cross-Project Synergy)

**5-1. 공유 가능 코드**
- 여러 프로젝트에서 비슷하게 구현된 기능 → 공통 모듈로
- 공통 UI 컴포넌트 추출 기회

**5-2. 통합 대시보드 기회**
- 여러 프로젝트 데이터를 한눈에 볼 수 있는 통합 뷰
- 크로스 프로젝트 검색/알림

**5-3. 인증/권한 통합**
- auth-worker 활용 확대 가능 여부
- SSO 적용 범위 확장

---

### Phase 6: 트렌드 기반 제안 (Trend-Based Suggestions)

**6-1. 기술 트렌드**
- 적용 가능한 최신 라이브러리/프레임워크
- Svelte 5 runes 활용 확대
- 새로운 Web API 활용 (예: View Transitions, Push Notifications)

**6-2. UX 트렌드**
- 다크모드 (미적용 프로젝트)
- PWA 기능 강화 (오프라인 지원)
- 모바일 최적화

**6-3. AI 활용 기회**
- LLM 연동 가능 영역
- 자동 분류/추천 기능
- 자연어 검색/명령

## 아이디어 평가 기준

각 아이디어는 아래 기준으로 우선순위 결정:

| 기준 | 높음 | 중간 | 낮음 |
|-----|------|------|------|
| **사용자 가치** | 일상적 사용, 핵심 흐름 개선 | 간헐적 사용, 편의 기능 | 드물게 사용, nice-to-have |
| **구현 난이도** | 1-4시간 | 1-3일 | 1주 이상 |
| **영향 범위** | 전체 사용자 | 특정 기능 사용자 | 일부 사용자 |
| **기술 부채** | 코드 품질 개선 | 중립 | 복잡도 증가 |

**우선순위 계산:**
- 높음: 사용자 가치 높음 + 구현 난이도 쉬움/중간
- 중간: 사용자 가치 중간 또는 구현 난이도 높음
- 낮음: 사용자 가치 낮음 또는 장기 프로젝트
