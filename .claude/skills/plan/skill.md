# 계획 문서 작성

사용자의 아이디어나 요구사항을 `common/docs/plan/`에 계획 문서로 정리합니다.

## 트리거

- "계획해", "plan", "아이디어", "기획"
- 새로운 기능이나 개선사항을 논의할 때

## 파일 위치

```
common/docs/plan/YYYY-MM-DD_{주제}.md
```

예: `common/docs/plan/2026-01-07_dark-mode.md`

## 실행 단계

### 1단계: 요구사항 파악

사용자에게 다음을 확인:
- 대상 프로젝트 (없으면 전체 공통)
- 구현하고 싶은 기능/개선사항
- 우선순위 (있다면)

### 2단계: 문서 생성

`common/docs/plan/` 에 아래 템플릿으로 문서 생성:

```markdown
# {제목}

> 작성일: YYYY-MM-DD
> 대상 프로젝트: {project-name | 공통}
> 상태: 검토 대기
> 진행률: 0/N (0%)

---

## 개요

{배경 및 목적 설명}

---

## 구현 항목

| 우선순위 | 항목 | 설명 | 난이도 |
|:-------:|------|------|:------:|
| P0 | {항목1} | {설명} | 낮음/중간/높음 |
| P1 | {항목2} | {설명} | ... |
| P2 | {항목3} | {설명} | ... |

---

## 기술적 고려사항

- {고려사항 1}
- {고려사항 2}

---

## 구현 순서

1. [ ] P0: {항목1}
2. [ ] P1: {항목2}
3. [ ] P2: {항목3}

---

*상태: 검토 대기 | 진행률: 0/N (0%)*
```

### 3단계: 안내

문서 생성 후 사용자에게 안내:

```
📝 계획 문서가 생성되었습니다!

파일: common/docs/plan/YYYY-MM-DD_{주제}.md

다음 단계:
- 검토 후 수정이 필요하면 말씀해주세요
- 구현을 시작하려면 "다음" 또는 "구현해"라고 말씀해주세요
```

## 문서 상태 & 진행률

| 상태 | 의미 |
|------|------|
| `검토 대기` | 아이디어 수집 중, 검토 필요 |
| `진행 중` | TODO로 이동된 항목 있음 |
| `완료` | 모든 항목 완료 |

**진행률 업데이트 규칙:**
- 문서 헤더와 푸터에 `진행률: 완료수/전체 (백분율%)` 표시
- 구현 순서의 `[x]` 개수를 세어 자동 계산
- 예: 3개 중 2개 완료 → `진행률: 2/3 (67%)`

## 우선순위 기준

| 우선순위 | 기준 |
|:-------:|------|
| P0 | 즉시 실행 가능, 핵심 기능 |
| P1 | 중요하지만 P0 이후 |
| P2 | 있으면 좋은 기능 |
| P3 | 장기 과제 |

## 난이도 기준

| 난이도 | 기준 |
|:-----:|------|
| 낮음 | 1-2시간, 단순 구현 |
| 중간 | 반나절, 여러 파일 수정 |
| 높음 | 하루 이상, 아키텍처 변경 |

## 환경

- **Windows**: 백슬래시(`\`), 절대경로, PowerShell 전용
