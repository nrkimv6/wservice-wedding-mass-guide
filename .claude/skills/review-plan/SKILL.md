# 계획서 재검토 → 확장 → 커밋

새로 생성된 계획서를 검증하고, `/expand-todo` 스킬의 실행 단계를 수행한 뒤 커밋합니다.
`/reflect` 완료 후 자동 호출되거나, 독립적으로 사용할 수 있습니다.

## 트리거

- "계획서 검토", "review-plan", "계획 검토"
- `/reflect` 완료 후 자동 호출 ("조사만" 모드 제외)

## 입력

- **필수**: 계획서 경로 1개 이상 (예: `docs/plan/2026-03-31_fix-xxx.md`)
- **선택**: 없음

`/reflect`에서 호출 시, reflect가 생성한 계획서 경로 목록이 자동 전달된다.
독립 호출 시: `review-plan {경로}` 형식으로 경로를 직접 지정한다.

## 실행 단계

### 1단계: 계획서 재검토

대상 계획서 각각에 대해 아래 3가지를 검증한다.

**A. side effect 체크:**
- 계획서에 명시된 수정 대상이 다른 모듈/기능에 영향 없는지
- import 참조, 공유 함수를 Grep으로 확인

**B. 목표 ↔ 예상결과 일관성:**
- 계획서의 "문제 → 목표 → 예상결과"가 논리적으로 연결되는지
- 목표 달성이 예상결과로 이어지는지 검증

**C. 기존 plan 중복 체크:**
- `docs/plan/` 디렉토리의 기존 계획서 제목과 비교
- 겹치면 경고 출력 + 기존 plan에 항목 추가 제안 (신규 생성 중단)

**재검토 실패 시:**
- 해당 계획서 파일을 삭제
- 실패 사유를 출력 (자동 재생성 안 함, 사용자 판단에 맡김)

### 2단계: expand-todo 수행

재검토 통과한 각 계획서에 대해 `/expand-todo` 스킬의 실행 단계를 수행한다.
- 코드베이스 분석 기반 2레벨 체크리스트 확장
- 정합성 검증 (V1~V6)
- expand-todo 스킬의 모든 규칙 적용 (테스트 Phase, 분리 판단 등)

### 3단계: 커밋

- 화이트리스트 파일만 스테이징: `docs/plan/*.md`, `TODO.md`
- 커밋 메시지: `docs: review — {계획서 제목 요약}`
- 커밋 스크립트 사용 (git commit 직접 사용 금지)

## 출력 형식

```markdown
## 계획서 재검토 결과

| # | 계획서 | 재검토 | expand | 비고 |
|---|--------|--------|--------|------|
| 1 | {파일명} | ✅ 통과 | ✅ {N}개 작업 | — |
| 2 | {파일명} | ❌ 실패 | — | {사유} |

구현을 시작하려면 "다음" 또는 "구현해"라고 말씀해주세요.
```

## 주의사항

- **코드 직접 수정 금지** — 계획서 검증과 확장만 수행
- **기존 plan 무단 수정 금지** — 중복 발견 시 사용자에게 보고만
- **expand-todo 규칙 준수** — 테스트 Phase, 정합성 검증 등 expand-todo의 모든 규칙 적용

## 환경

- **Windows**: 백슬래시(`\`), 절대경로, PowerShell 전용
- **커밋**: 커밋 스크립트 필수 (`commit.ps1` 또는 `commit.sh`)
